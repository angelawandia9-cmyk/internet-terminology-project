Quick Reference Table

A summary of all 16 Internet terminology terms — their roles, definitions and how they work.

# Term Role Definition How It Works
01 World Wide Web Networking A global system of interlinked hypertext documents accessed via the Internet using web browsers. Browsers send HTTP requests to servers which return HTML pages that are rendered and displayed to users.
02 TCP/IP Protocols The suite of communication protocols that define how data is transmitted across networks and forms the backbone of the Internet. IP routes data packets between devices while TCP ensures all packets arrive correctly and reassembles them in order.
03 Web Browsers Client Software Software applications that retrieve, render, and display web pages from the Internet for users to interact with. The browser sends HTTP requests, receives HTML/CSS/JS files from the server, and renders them as visual pages on screen.
04 Web Servers Server Infrastructure Hardware or software systems that store web content and deliver it to clients over HTTP or HTTPS upon request. Listens for HTTP requests on port 80 or 443, locates the requested resource, and sends it back with an HTTP status code.
05 URL Addressing Uniform Resource Locator — the standardised address used to locate a specific resource on the Internet. The browser reads the protocol, asks DNS to resolve the domain, connects to the server, and requests the resource at the specified path.
06 DNS Naming System Domain Name System — the Internet's directory that translates human-readable domain names into numerical IP addresses. Queries pass through resolver, root, TLD, and authoritative servers in a hierarchy until the correct IP address is returned.
07 HTTP Protocol Hypertext Transfer Protocol — the protocol that governs how data is requested and transmitted between browsers and web servers. Client sends a request with a method such as GET or POST, and the server responds with a status code and the requested content.
08 Intranet Private Networks A private network that uses Internet technologies to share information and resources securely within an organisation. Uses firewalls and VPNs to restrict access to authorised users only, hosting internal tools and portals not visible publicly.
09 Extranet Extended Networks A controlled extension of an intranet that grants limited access to approved external parties such as suppliers and partners. External users connect via VPN or a secure portal where the firewall verifies credentials and grants access to permitted resources only.
10 Multitier Architecture System Design A software design pattern that separates an application into distinct logical layers each with specific responsibilities. The browser (Presentation) sends requests to the app server (Logic) which queries the database (Data) and returns results back up the chain.
11 FTP File Transfer File Transfer Protocol — a standard network protocol used to transfer files between a client and a server over a TCP/IP network. Uses port 21 for commands and port 20 for data transfer. Users authenticate then upload or download files between client and server.
12 HTML Markup Language Hypertext Markup Language — the standard language used to create and structure content on the World Wide Web. Uses tags to define structure and content. The browser parses the HTML to build a DOM tree which is then rendered as a visual page.
13 Web 1.0 Web Evolution The first generation of the Web from 1991 to 2004, characterised by static read-only pages with no user interaction. Webmasters hand-coded HTML pages that users could only read and navigate. No accounts, comments, or user-generated content existed.
14 Web 2.0 Web Evolution The second generation of the Web from 2004 onwards, characterised by user-generated content, social networking, and interactivity. Platforms like YouTube and Facebook allow users to create accounts, post content, comment, and interact using AJAX and APIs.
15 Web 3.0 Web Evolution The emerging third generation of the Web focused on decentralisation, user data ownership, blockchain, and artificial intelligence. Blockchain powers dApps and smart contracts while AI enables semantic understanding, giving users ownership and control of their data.
16 Web 4.0 Web Evolution The predicted fourth generation of the Web — a symbiotic intelligent web integrating physical and digital worlds through AI and IoT. AI agents, IoT sensors, and ambient computing work together so the digital world responds to human needs without explicit commands.
